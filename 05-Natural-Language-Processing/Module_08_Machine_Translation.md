## 📌 Chapter 1：SMT 数学基础（面试高频推导）

### 核心目标

$$t^* = \arg\max_t P(t|s)$$

### 为什么不直接算 $P(t|s)$？

```
直接建模 P(t|s) 的问题：
  "I love NLP" 的中文翻译有无数种
  → 训练数据无法覆盖所有 (s,t) 组合
  → 难以建模，太发散
```

### 贝叶斯变换（核心推导）

$$P(t|s) = \frac{P(s|t) \cdot P(t)}{P(s)}$$

因为 $P(s)$ 是给定常量，可忽略：

$$\boxed{t^* = \arg\max_t \underbrace{P(s|t)}_{\text{翻译模型}} \cdot \underbrace{P(t)}_{\text{语言模型}}}$$

| 模型 | 作用 | 问它的问题 | 如何构建 |
|---|---|---|---|
| **翻译模型 P(s\|t)** | 确保意思翻对 | 给定目标语言 t，能还原成源语言 s 吗？ | 词对齐技术 |
| **语言模型 P(t)** | 确保目标语言通顺 | 这句话像不像正常人说的？ | N-gram / 神经 LM |

> **记忆口诀**：翻译模型管"对不对"，语言模型管"顺不顺"

### 为什么 P(s|t) 比 P(t|s) 更好估计？

```
反向建模 P(s|t)：给定中文 → 可能的英语来源较少，更确定
正向建模 P(t|s)：给定英语 → 可能的中文翻译无数种，太发散
```

### 计算示例

```
源句子 s："I love NLP"

候选1: t₁ = "我爱自然语言处理"
  P(s|t₁)=0.8 × P(t₁)=0.9 = 0.72 ← 最高！

候选2: t₂ = "我喜欢 NLP"
  P(s|t₂)=0.6 × P(t₂)=0.7 = 0.42

候选3: t₃ = "爱我 NLP"（语法错误）
  P(s|t₃)=0.3 × P(t₃)=0.1 = 0.03
```

## 📌 Chapter 2：Seq2Seq 架构

### 核心思想

```
编码器：读取整个输入序列 → 压缩成"语义向量"（上下文向量）
解码器：从语义向量出发 → 逐步生成输出序列
```

### 应用场景

| 任务 | 输入序列 | 输出序列 |
|---|---|---|
| 机器翻译 | "I love NLP" | "我爱自然语言处理" |
| 文本摘要 | 长文章 | 摘要 |
| 对话系统 | "你好吗？" | "我很好，谢谢！" |

### 上下文向量的三重身份

| 身份 | 含义 |
|---|---|
| **语义向量** | 压缩了输入句子的全部意思 |
| **Thought Vector** | Hinton 的叫法："思想的数学表示" |
| **信息瓶颈** | ⚠️ 最大的问题：固定长度，长句信息会丢失 |

### 解码器公式

$$c = h_n^e, \quad h_0^d = c$$
$$h_t^d = g(\hat{y}_{t-1}, h_{t-1}^d, c)$$
$$\hat{y}_t = \text{softmax}(h_t^d)$$

## 📌 Chapter 3：Attention 在 MT 中的应用

### 动机：打破信息瓶颈

```
问题：长句子 → 压缩成一个向量 → 信息丢失
解决：解码每个词时，动态生成新的上下文向量
     c_t = Σ α_i · h_i^e（加权所有编码器隐藏状态）
```

### Attention 权重计算

$$\alpha_{ij} = \frac{\exp(\text{score}(h_{i-1}^d, h_j^e))}{\sum_k \exp(\text{score}(h_{i-1}^d, h_k^e))}$$

$$c_i = \sum_j \alpha_{ij} h_j^e$$

**解码器更新**：
$$s_t = \text{RNN}(s_{t-1}, y_{t-1}, c_t)$$

**Attention 的四大优势**：
- ✅ 解决信息瓶颈：不需要把所有信息压缩到一个向量
- ✅ 处理长句子：每次都能看到完整输入
- ✅ 可解释性：可以看到模型关注哪些词
- ✅ 提高翻译准确性：特别是长句子

**教授核心观点**：
> "一切都是关于分布：如何移动和改变分布，如何组合不同的分布。"

## 📌 Chapter 4：Beam Search

### 贪婪搜索的问题

```
贪婪：每步选最高概率的词
  "我"(0.8) → "喜欢"(0.7) → "你"(0.6) = 0.336

更好的路径：
  "我"(0.8) → "爱"(0.6) → "你"(0.9) = 0.432 ← 更高！
```

### Beam Search 流程（Beam Width = k）

```
Step 1：选前 k 个词作为初始候选
Step 2：每个候选扩展出所有可能的下一词（k × |V| 个）
Step 3：从中选 k 个总概率最高的保留
Step 4：重复直到所有候选生成 </s>
Step 5：从 k 个完整路径中选最优
```

**注意**：Beam Search 偏好较短序列，需要**长度归一化**修正：

$$\text{score\_normalized} = \frac{\log P(\text{sequence})}{\text{length}^\alpha}$$

### 策略对比

| 策略 | 每步保留 | 速度 | 质量 | 适用场景 |
|---|---|---|---|---|
| **贪婪搜索** | 1 | 最快 | 可能次优 | 实时/低延迟 |
| **Beam Search** | k | 较快 | 更好 | 翻译/摘要 |
| **Top-k 采样** | 随机前k | 快 | 多样 | 创意生成 |
| **Top-p 采样** | 随机前p% | 快 | 自然 | 对话 |

---

### 🎯 产品经理要掌握到：

✅ **Beam Width 是影响翻译质量和速度的关键超参数**：k 越大，质量越好，速度越慢，成本越高

✅ **长度归一化是产品细节**：不加的话模型偏向输出短答案，影响用户体验

✅ **不同生成策略对应不同产品场景**：翻译要准确用 Beam Search，聊天要自然用 Top-p

---

### 📐 算法工程师要掌握到：

✅ 能实现 Beam Search（包括长度归一化）

✅ 知道现代 LLM 推理时的选择：Beam Search（确定性强）vs 采样（多样性高）

## 📌 Chapter 5：MT 的挑战与伦理

| 挑战 | 具体问题 | 技术应对 |
|---|---|---|
| **词对齐** | 源语言和目标语言词数/词序不同 | Attention 动态对齐 |
| **无对应词** | 文化特有概念（"缘分"）无法直接翻译 | 上下文嵌入、意译 |
| **性别偏见** | 中性语言翻译成英语时默认男性代词 | 数据去偏、多元化训练数据 |
| **数据偏见** | 训练数据的偏见被模型放大 | 审查训练数据、对抗训练 |

### 回译（Back Translation）

> 用目标语言的单语数据反向翻译成源语言，生成合成平行语料，提高翻译质量。
> **产品价值**：低成本扩充训练数据

---

### 🎯 产品经理要额外关注（伦理视角）：

✅ **AI 翻译的偏见是产品风险**：医疗/法律翻译中的偏见可能造成严重后果

✅ **理解"语法正确性"的统计本质**：SMT 不强制语法规则，训练数据决定输出质量

> **产品面试常问**：
> - "你们的翻译产品如何处理文化差异？"
> - "如何检测和减少翻译中的偏见？"

## ⚡ 踩坑 & 经验

1. **P(s|t) 不是"反向翻译"**：翻译方向始终是 s→t，P(s|t) 只是贝叶斯分解后的计算工具
2. **上下文向量的信息瓶颈是 Attention 诞生的根本原因**：理解这个因果关系
3. **Beam Width 不是越大越好**：太大会导致生成太保守，缺乏多样性
4. **"一切都是分布"**：教授的核心思想，注意力机制本质上是对分布做加权组合

## 🔖 核心公式速查卡

$$t^* = \arg\max_t P(s|t) \cdot P(t) \quad \text{（SMT目标）}$$

$$\alpha_{ij} = \text{softmax}(\text{score}(h_{i-1}^d, h_j^e)) \quad \text{（注意力权重）}$$

$$c_i = \sum_j \alpha_{ij} h_j^e \quad \text{（动态上下文向量）}$$

$$\text{Attention}(Q,K,V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V \quad \text{（自注意力）}$$
